# Start Here

Эта страница — стартовый экран архитектуры. Она помогает выбрать первый шаг.

## Что это такое

AI Dev Architecture Template — это набор правил и файлов памяти для работы с AI-агентом.

Простая аналогия: это рабочий журнал проекта.

- `AGENTS.md` и `CLAUDE.md` — короткие инструкции для AI-агента.
- `ai/current-task.md` — задача, которую агент выполняет сейчас.
- `ai/paused-tasks.md` — незавершённые задачи, поставленные на паузу.
- `ai/future-tasks.md` — идеи на потом.
- `ai/project-context.md` — описание проекта: стек, команды, важные правила.
- `ai/skills/*/SKILL.md` — пошаговые процедуры для агента.

## Выберите ваш сценарий

### 1. Я подключаю архитектуру к новому проекту

Самый простой путь — [универсальный стартовый промт](../prompt/README.md): скопируйте его в AI-агента в папке проекта, он установит архитектуру сам.

Ручной способ — [инструкция по установке](install.md).

После установки попросите агента:

```text
Run environment-check.
```

Потом поставьте первую задачу. Агент должен использовать `task-intake` и записать её в `ai/current-task.md`.

### 2. У меня уже есть проект со старой версией архитектуры

Самый простой путь — тот же [универсальный стартовый промт](../prompt/README.md): он сам определит, что архитектура уже установлена, и предложит обновление.

Ручной способ — [инструкция по обновлению установленной архитектуры](update-installed-projects.md).

Сначала запустите проверку версии и dry-run. Dry-run — это пробный запуск: он показывает, что изменится, но не меняет файлы.

### 3. Я хочу поставить первую задачу

Используйте такой промт:

```text
Сначала сделай environment-check.
Потом используй task-intake и запиши эту работу в ai/current-task.md.

Задача:
<опишите, что нужно сделать>

Как понять, что готово:
<опишите видимый результат>
```

Если `ai/current-task.md` пустой, агент должен записать туда задачу. Если там уже есть незавершённая задача, агент должен предложить `task-switch`.

### 4. У меня баг или сложная задача

Используйте Superpowers.

Superpowers — это внешний плагин/методология для сложной разработки: он помогает сначала продумать задачу, написать план, проверять гипотезы и не чинить баги наугад.

Баги, регрессии, падения, flaky behavior, performance-проблемы, крупные изменения, миграции и задачи с непонятным радиусом изменений должны идти через Superpowers, если он доступен.

Если Superpowers не установлен, агент должен сказать об этом и спросить: установить/настроить его или продолжить вручную.

### 5. Я хочу закрыть задачу

Попросите:

```text
Run task-finish.
```

Закрытие задачи теперь означает два действия:

1. Проверить, что задача действительно готова.
2. Сохранить результат.

Если GitHub подключён, агент должен сделать commit и push. Commit — это сохранённая точка истории в Git. Push — отправка этой точки на GitHub.

Если GitHub не подключён, используйте [режим без GitHub](no-github.md).

### 6. У меня нет GitHub

Откройте [режим без GitHub](no-github.md).

Архитектура может работать локально. В этом случае результат сохраняется на компьютере через локальный Git commit или, если Git недоступен, через patch/archive fallback.

## Удаление архитектуры

Сначала прочитайте [безопасную инструкцию по удалению](uninstall.md). Не удаляйте `ai/`, `AGENTS.md`, `CLAUDE.md`, `.claude/` или `.codex/` целиком без проверки содержимого.

## Что агент должен делать сам

- Перед первой задачей запускать `environment-check`.
- Перед реальной работой запускать `task-intake`.
- Не перезаписывать незавершённую задачу без `task-switch`.
- Не закрывать задачу без `task-finish`.
- Для багов и сложных задач использовать Superpowers, если он доступен.

## Что агент не должен делать сам

- Устанавливать внешние плагины без подтверждения.
- Запускать все пункты меню после `environment-check`.
- Перезаписывать protected architecture files без `architecture-update`.
- Считать задачу закрытой, если результат не сохранён.
