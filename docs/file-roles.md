# Роли файлов

## Шаблонные файлы

Эти файлы копируются из шаблона и обычно одинаковы для разных проектов:

- `AGENTS.md`
- `CLAUDE.md`
- `ai/architecture.md`
- `ai/skills/*/SKILL.md`

## Проектные файлы

Эти файлы нужно заполнять отдельно для каждого проекта:

- `ai/project-context.md`
- `ai/current-task.md`
- `ai/decisions.md`
- `ai/changelog.md`

## AGENTS.md

Входной файл для Codex.

Он должен быть коротким и универсальным.

Не нужно хранить в нём полные проектные правила.

## CLAUDE.md

Входной файл для Claude Code.

По смыслу должен совпадать с `AGENTS.md`.

## ai/project-context.md

Постоянный контекст конкретного проекта.

В нём хранятся:

- стек;
- команды;
- экраны или модули;
- модель данных;
- инварианты проекта;
- хрупкие зоны.

## ai/current-task.md

Только текущая задача.

Этот файл часто меняется.

## ai/decisions.md

Только важные активные решения.

Используй его для решений, которые будущие агенты не должны случайно сломать.

## ai/changelog.md

Последние заметные изменения.

Храни только последние 2–4 недели.

## ai/skills/*/SKILL.md

Переиспользуемые процедуры.

Не нужно загружать все skills сразу. Используй только те, которые нужны для задачи.

## Дополнительные внешние инструменты

## environment-check

`ai/skills/environment-check/SKILL.md` нужен для первого запуска или проверки установки архитектуры.

Он проверяет наличие обязательных файлов и skills.

Он не должен запускаться в каждой обычной задаче.

## ai/external-tools.md

Файл со списком ожидаемых внешних skills и инструментов.

Он нужен, чтобы environment-check понимал, что проверить при первом запуске.

Внешние skills и инструменты не входят в шаблон напрямую. Если их нет, агент сообщает об этом, но не считает это критической ошибкой.

Ожидаемые внешние инструменты:
- code-review-graph
- agent-skills-for-context-engineering
- claude-seo

## Приоритет внешних инструментов

Ожидаемые внешние инструменты:
- code-review-graph
- agent-skills-for-context-engineering
- claude-seo

Controlled external methodologies:
- Superpowers

Optional alternatives:

`code-review-graph` — основной инструмент для code review и blast-radius analysis.

`Superpowers` проверяется при первом запуске, но не используется по умолчанию. Он включается только после явного разрешения пользователя или если в `ai/current-task.md` указано `Use Superpowers: yes`.


## ai/paused-tasks.md

Файл для задач, которые временно поставили на паузу через `task-switch`.

Это не backlog и не список идей.

Используй только для незавершённых задач, к которым нужно вернуться.


## task-switch

`ai/skills/task-switch/SKILL.md` нужен для безопасного переключения между незавершёнными задачами.

Он защищает `ai/current-task.md` от случайной перезаписи.

Он не закрывает задачу. Для закрытия используется `task-finish`.

Он срабатывает, когда новый запрос меняет цель, work mode, основные файлы, Done criteria или создаёт отдельный результат.
