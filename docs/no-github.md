# Режим без GitHub

GitHub удобен, но архитектура может работать и без него.

## Что меняется без GitHub

GitHub — это удалённое место, куда обычно отправляют историю изменений проекта.

Если GitHub нет, архитектура всё равно хранит контекст в файлах:

- `ai/current-task.md`
- `ai/project-context.md`
- `ai/decisions.md`
- `ai/changelog.md`
- `ai/future-tasks.md`
- `ai/paused-tasks.md`

Но результат работы нужно сохранять локально.

## Лучший вариант: локальный Git

Git — это история изменений на вашем компьютере.

Если Git доступен, закрытие задачи должно делать локальный commit.

Commit — это точка сохранения: можно посмотреть, какие файлы изменились, и вернуться к прошлому состоянию.

Команды обычно такие:

```bash
git status
git add .
git commit -m "описание задачи"
```

В этом режиме агент должен ясно сказать:

```text
Saved locally, not pushed to GitHub.
```

## Если Git тоже недоступен

Тогда используйте слабый, но рабочий fallback:

- patch-файл с изменениями;
- zip/archive копию проекта;
- список изменённых файлов и краткое описание.

Patch — это файл со списком изменений. Его можно применить позже, но он менее удобен, чем Git.

## Как закрывать задачу без GitHub

Попросите агента:

```text
Run task-finish in local-only mode.
```

Агент должен:

1. Проверить Done criteria.
2. Обновить task memory после подтверждения.
3. Создать локальный commit, если Git доступен.
4. Если Git недоступен, создать patch или archive fallback.
5. Сказать, где сохранён результат.

## Когда лучше всё-таки подключить GitHub

GitHub полезен, если:

- вы работаете на нескольких компьютерах;
- хотите иметь резервную копию;
- хотите, чтобы другой человек мог посмотреть изменения;
- хотите безопасно откатываться к прошлым версиям.

Но для одиночной локальной работы архитектура может стартовать без GitHub.
