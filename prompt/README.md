# Универсальный стартовый промт

Один промт для двух случаев: **установить** архитектуру в проект или **обновить** уже установленную. Агент сам определит, что нужно.

Как пользоваться:

1. Откройте AI-агента (Claude Code, Codex или другой) в папке вашего проекта.
2. Скопируйте промт ниже целиком и отправьте агенту.
3. Отвечайте на вопросы агента. Он ничего не перезапишет без вашего подтверждения.

```text
Ты работаешь в моём проекте как AI coding agent.

Подключи или обнови AI-архитектуру разработки из репозитория:
https://github.com/zykovsrg/ai-dev-architecture-template

Шаг 1. Определи ситуацию.

Проверь, есть ли в корне проекта AGENTS.md, CLAUDE.md и папка ai/.
- Если их нет — это первая установка. Иди в Шаг 2.
- Если они есть — это обновление. Иди в Шаг 3.
- Если есть только часть файлов — покажи, что нашёл, и спроси меня, как поступить.

Шаг 2. Первая установка.

1. Склонируй шаблон во временную папку:
   git clone https://github.com/zykovsrg/ai-dev-architecture-template.git /tmp/ai-dev-arch-template
2. Скопируй файлы шаблона в проект, не перезаписывая существующие:
   rsync -av --ignore-existing /tmp/ai-dev-arch-template/template/ ./
3. Покажи список добавленных файлов.
4. Если каких-то обязательных файлов не хватает, назови их и предложи восстановить из шаблона.
5. Иди в Шаг 4.

Шаг 3. Обновление.

1. Скачай updater и сначала только проверь версию, ничего не меняя:
   curl -fsSL https://raw.githubusercontent.com/zykovsrg/ai-dev-architecture-template/main/scripts/update-installed-architecture.sh -o /tmp/update-arch.sh
   bash /tmp/update-arch.sh --check
2. Если обновление есть, покажи превью изменений и объясни его простыми словами:
   bash /tmp/update-arch.sh --dry-run
3. Применяй обновление только после моего явного подтверждения:
   bash /tmp/update-arch.sh --apply --commit
4. Updater обновляет только файлы правил (AGENTS.md, CLAUDE.md, ai/architecture.md, ai/external-tools.md, ai/skills/*). Файлы памяти проекта (ai/current-task.md, ai/paused-tasks.md, ai/future-tasks.md, ai/project-context.md, ai/decisions.md, ai/changelog.md) он не перезаписывает. Убедись, что это так, прежде чем применять.
5. Иди в Шаг 4.

Шаг 4. После установки или обновления.

1. Выполни environment-check и покажи, что на месте, чего не хватает и какие команды доступны дальше. Список команд — это меню, не запускай их автоматически.
2. Для нового проекта предложи заполнить ai/project-context.md и поставить первую задачу через task-intake.
3. Для существующего проекта покажи текущую задачу из ai/current-task.md, если она есть.

Правила:

- Общайся со мной на русском. Я не разработчик — объясняй технические термины простыми словами.
- Ничего не перезаписывай и не удаляй без моего подтверждения.
- Не меняй код приложения.
- Не устанавливай ничего дополнительно без моего подтверждения.
- Protected architecture files меняй только через architecture-update после моего явного подтверждения.
- Файлы памяти проекта меняй только через подходящий workflow.
```
