# Архитектура для ии-разрабокти
И работы над задачами со сложным контекстом.

Эта архитектура — набор промтов, файлов и скриптов, которые помогают вести долгосрочные проекты.

Архитектура помогает:

   - фиксировать важные решения в проекте;
   - фокусироваться на важной задаче, доводить её до конца, не переключаться от задачи к задаче;
   - планировать работу;
   - легко переключаться между разными ИИ-агентами без потери контекста и объяснения требований заново;
   - тратить меньше токенов;
   - не копить технический долг.

В этом репозитории — готовый шаблон, который ставится в папку проекта локально или в репозиторий, если вы работаете в облаке. Архитектура предназначена для работы через «Клод Код» или «Кодекс».

## Как работает архитектура

Архитектура фиксирует контекст в отдельные файлы. Благодаря этому сессии и агенты можно менять на лету без пересказа деталей задачи. А текущая задача, будущие идеи и принятые решения лежат в разных файлах и не смешиваются.

Нужный контекст в архитектуре подключается только при необходимости. Это помогает держать фокус ИИ-агента на главном. Ещё так агент тратит меньше токенов, так как читает только нужные файлы, а не весь проект каждый раз.

## Как строится работа с задачами

1. Вы ставите задачу.
2. ИИ-агент фиксирует задачу и критерии, по которым она считается выполненной.
3. Когда критерии соблюдены — задача считается готовой и закрывается.
4. Если задача не готова, а пользователь просит выполнить другую задачу, текущая задача ставится на паузу. Затем агент создаёт новую задачу.

## Режимы работы архитектуры

Четыре режима работы архитектуры:

   - `review` — только анализ без работы с кодом или файлами;
   - `implementation` — работа с кодом или файлами;
   - `task-finish` — ИИ-агент закрывает задачу, пишет историю изменений, если есть параллельный репозиторий — обновляет;
   - `architecture-update` — обновление архитектуры: работа с файлами архитектуры.

Архитектура одновременно работает только в одном режиме. Это нужно, чтобы держать фокус и не расширять скоуп в моменте.

## Установка в проект: стартовый промт

Один промт для двух случаев: установить архитектуру в проект или обновить уже установленную. Агент сам определит, что нужно.

Как пользоваться:

1. Откройте AI-агента (Claude Code, Codex или другой) в папке вашего проекта.
2. Скопируйте промт ниже целиком и отправьте агенту.
3. Отвечайте на вопросы агента. Он ничего не перезапишет без вашего подтверждения.

```text
Ты работаешь в моём проекте как AI coding agent.

Подключи или обнови AI-архитектуру разработки из репозитория:
https://github.com/zykovsrg/ai-dev-architecture-template

Шаг 1. Определи ситуацию.

Проверь, есть ли в корне проекта AGENTS.md, CLAUDE.md и папка ai/.
- Если их нет — это первая установка. Иди в Шаг 2.
- Если они есть — это обновление. Иди в Шаг 3.
- Если есть только часть файлов — покажи, что нашёл, и спроси меня, как поступить.

Шаг 2. Первая установка.

1. Склонируй шаблон во временную папку:
   git clone https://github.com/zykovsrg/ai-dev-architecture-template.git /tmp/ai-dev-arch-template
2. Скопируй файлы шаблона в проект, не перезаписывая существующие:
   rsync -av --ignore-existing /tmp/ai-dev-arch-template/template/ ./
3. Покажи список добавленных файлов.
4. Если каких-то обязательных файлов не хватает, назови их и предложи восстановить из шаблона.
5. Иди в Шаг 4.

Шаг 3. Обновление.

1. Скачай updater и сначала только проверь версию, ничего не меняя:
   curl -fsSL https://raw.githubusercontent.com/zykovsrg/ai-dev-architecture-template/main/scripts/update-installed-architecture.sh -o /tmp/update-arch.sh
   bash /tmp/update-arch.sh --check
2. Если обновление есть, покажи превью изменений и объясни его простыми словами:
   bash /tmp/update-arch.sh --dry-run
3. Применяй обновление только после моего явного подтверждения:
   bash /tmp/update-arch.sh --apply --commit
4. Updater обновляет только файлы правил (AGENTS.md, CLAUDE.md, ai/architecture.md, ai/external-tools.md, ai/skills/*). Файлы памяти проекта (ai/current-task.md, ai/paused-tasks.md, ai/future-tasks.md, ai/project-context.md, ai/decisions.md, ai/changelog.md) он не перезаписывает. Убедись, что это так, прежде чем применять.
5. Иди в Шаг 4.

Шаг 4. После установки или обновления.

1. Выполни environment-check и покажи, что на месте, чего не хватает и какие команды доступны дальше. Список команд — это меню, не запускай их автоматически.
2. Для нового проекта предложи заполнить ai/project-context.md и поставить первую задачу через task-intake.
3. Для существующего проекта покажи текущую задачу из ai/current-task.md, если она есть.

Правила:

- Общайся со мной на русском. Я не разработчик — объясняй технические термины простыми словами.
- Ничего не перезаписывай и не удаляй без моего подтверждения.
- Не меняй код приложения.
- Не устанавливай ничего дополнительно без моего подтверждения.
- Protected architecture files меняй только через architecture-update после моего явного подтверждения.
- Файлы памяти проекта меняй только через подходящий workflow.
```

## Из чего состоит архитектура

Три сущности: входные файлы, файлы памяти, инструменты: скиллы + плагины.

1. Входные файлы — ключевые правила. Их ИИ-агент читает при каждом вашем запросе к языковой модели:

   - `AGENTS.md` — входной файл для Codex.
   - `CLAUDE.md` — входной файл для Claude Code.

2. Файлы памяти проекта — их агент читает по необходимости:

- `ai/project-context.md` — что за проект: стек, структура, хрупкие места.
- `ai/decisions.md` — важные долгосрочные решения.
- `ai/current-task.md` — текущая задача.
- `ai/paused-tasks.md` — задачи, поставленные на паузу через `task-switch`.
- `ai/future-tasks.md` — идеи и будущие задачи вне текущей работы.
- `ai/changelog.md` — последние изменения.
- `ai/external-tools.md` — дополнительные инструменты, плагины и методологии.

3. Скиллы — промты, которые применяются только в нужном контексте. Основные скиллы:

Стартовая проверка:
- `start-screen` — стартовый экран.
- `environment-check` — проверяет, всё ли нужное установлено для работы архитектуры.

Работа с задачами:
- `task-intake` — принимает новую задачу.
- `task-switch` — безопасно переключает незавершённые задачи.
- `task-finish` — проверяет, можно ли закрыть задачу, и чистит контекст после подтверждения.

Работа с архитектурой:
- `architecture-update` — обновляет архитектуру, если пользователь явно попросит.

## Superpowers

Это плагин, который важно установить для полноценной работы архитектуры. Он помогает отлавливать баги, регрессии, краши и решать сложные задачи.

Superpowers — это набор методологий, которые обычно используют в разработке. Но их можно применять и для проектов, не связанных с кодом.

Вот основные методологии:

1. TDD (Test-Driven Development) — сначала пишем тест на нужное поведение, он падает, потом пишем код, чтобы тест прошёл. Гарантирует, что фича проверена, а не просто написана на глаз.

2. Систематическая отладка. Баги не чинятся наугад, ИИ-агент двигается шаг за шагом: воспроизвести баг → локализовать причину → понять, почему так случилось → починить.

3. Написание плана реализации для сложных многошаговых задач. Сначала пишется письменный план с контрольными точками, и только потом идёт выполнение. Код-ревью — обязательный финальный шаг. ИИ-агент просит проверить работу по чек-листу или провести тесты.

4. Изоляция экспериментов — рискованную или исследовательскую работу ИИ-агент ведёт в отдельной рабочей копии кода, не трогая основную ветку.

5. Параллельная работа через субагентов — если задача распадается на независимые куски, их можно передать нескольким агентам для одновременной работы.

## Структура репозитория

```text
template/   — то, что копируется в ваш проект (AGENTS.md, CLAUDE.md, ai/);
getting-started/ — стартовый экран;
docs/       — документация: термины, установка, обновление, сценарии;
scripts/    — файлы скриптов: install.sh, update-installed-architecture.sh, check-consistency.sh.
```
