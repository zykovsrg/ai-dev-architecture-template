description: |
  Use when reviewing interface text, button labels, empty states, tooltips, headings, error messages, or user-facing messages.
  Activates when:
  - the task changes text visible to the user
  - the user asks whether wording is clear, simple, short, or understandable
  - labels, empty states, tooltips, headings, or error messages may confuse the user
  Does NOT activate for:
  - backend logic
  - architecture decisions
  - visual layout without text changes
---

# Copy Review

Check:

1. Is the text clear?
2. Is the action obvious?
3. Is the wording short?
4. Is there unnecessary technical language?
5. Does the text match the product state?
6. Is the tone calm and direct?

Prefer simple interface language.
